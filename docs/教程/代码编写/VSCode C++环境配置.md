这个教程将带你熟悉 VS Code 的使用，结果是成功运行一个最简单的C++程序！

# 文件

在 VS Code 界面的左上角，软件图标的右侧，有一个"文件"菜单，点击它，你可以看到有许多选择。

## 打开文件

这个选项会让你选择文件系统中的一个文件，打开后会显示在主界面中，正常情况下就可以查看这个文件的内容，以及编辑这个文件了。

## 打开文件夹

这个选项会让你选择文件系统中的一个文件夹，打开后会将这个文件夹作为项目的根目录 (项目的最顶层)，之后你就可以在资源管理器中选择并查看这个文件夹下的所有文件和子文件夹。

# 查看

在"文件"菜单的右侧，有一个"查看"菜单，点击它，你可以看到有许多选择。

## 问题

这个选项会在主界面下方打开一个"问题"界面，如果编写的C++文件有错误，那么错误信息大概率会显示在这个界面。

## 终端

这个选项会在主界面下方打开一个"终端"界面，你可以在其中输入命令。

# C++环境的配置

一个 VS Code C++项目，包含 .vscode 文件夹下的配置文件，源文件和头文件。

对于 .vscode 文件夹，可以参考教程中的 .vscode 示例，其中对配置如何编写做出了详细的描述。

接下来，你可以查看教程中的 CppDemo 文件夹，你只需替换配置文件中的部分内容，然后在 VS Code 中打开 CppDemo文件夹，即可尝试运行一个最简单的C++程序。

如果你是 Windows 系统，一定要去安装 Visual Studio，否则C++项目无法运行！(https://visualstudio.microsoft.com/zh-hans/)
运行 Visual Studio Installer 之后，选择安装 Visual Studio Community，至少选择并点击"使用C++的桌面开发"进行安装。
安装完成后，你的头文件位置大概在"C:/Program Files/Microsoft Visual Studio/18/Insiders/VC/Tools/MSVC"/xx.xx.xxxxx/include"，如果有差异，请自行寻找调整。

## 如何运行

在"文件"菜单的右侧，有一个"运行"菜单，点击它，你可以看到有许多选择。

如果选择"以非调试模式运行"，会直接编译并运行程序，或者无法正常运行时，按下 Crtl + Shift + B 构建任务，再尝试运行。
也可以在终端中输入命令"./build/test"查看运行结果。

如果选择"启动调试"，会进入调试模式，并使程序触发了第一个断点时停下。

如果你点击"以非调试模式运行"或在终端中输入命令"./build/test"后在终端界面看到了"Hello World!"这条消息，说明你配置成功了

## 断点

你可以在文件编辑界面看到每一行文字左侧有行号，如果你点击C++文件中行号的左侧，会看到这行出现了一个红色的点，如果此时你点击"以非调试模式运行"，程序会在断点处停下来。

对于如何调试，你可以搜索或求助AI。例如"VS Code 如何调试 C++代码"。

现在你就可以在 CppDemo 的 Main.cpp 中的第5行添加一个断点，以非调试模式运行查看效果。

你也可以搜索相关资料或询问AI以了解断点的使用。
